# VisionMaster + AI 助手 Demo 作品报告

## 一、Demo 功能简述与完成工作

本作品是一个面向机器视觉应用开发场景的综合型 Demo。它以海康机器人 VisionMaster 平台为基础，将传统视觉方案加载、流程运行、图像显示、参数配置与 AI 助手能力集成到同一个 Windows 桌面程序中，目标是让开发者在调试视觉流程时能够更高效地完成方案运行、问题排查、结果分析和报告导出。

Demo 的核心功能包括：

1. **VisionMaster 方案接入**
   - 支持选择并加载 `.sol` 视觉方案文件。
   - 支持保存当前方案。
   - 加载方案后自动读取流程列表，开发者可以在界面中选择目标流程。

2. **流程运行与显示**
   - 支持单次运行流程。
   - 支持连续运行与停止连续运行。
   - 支持切换到图像渲染视图，观察流程运行结果。
   - 支持切换到参数配置视图，直接调整 VisionMaster 流程参数。

3. **日志与运行反馈**
   - 程序右侧提供日志显示区域，记录加载、运行、异常等关键信息。
   - 日志会同步写入本地 `Log` 目录，方便后续追溯。

4. **嵌入式 AI 助手**
   - 在主界面右下角集成 AI 聊天面板。
   - 支持配置 DeepSeek 或 OpenAI-compatible 接口。
   - 支持流式输出、停止回复、清空上下文、导出聊天记录。
   - 开发者可以在调试视觉流程时直接向 AI 提问，例如 SDK 接口含义、报错原因、参数配置建议等。

5. **VisionMaster 手册 RAG 增强问答**
   - Demo 内置 VM 手册检索增强能力。
   - 系统会将本地 VisionMaster CHM 手册解析、切片、建立索引。
   - 当用户向 AI 提问时，程序会自动检索相关手册片段，并将其作为上下文加入模型请求中。
   - 相比普通聊天机器人，该功能能让 AI 回答更贴近 VisionMaster 平台本身。

6. **结果统计与 AI 分析**
   - 连续运行结束后，`结果统计` 按钮可用。
   - 点击后弹出结果统计窗口，展示样本类别数量和占比。
   - 统计窗口包含柱状图、扇形图和明细表。
   - 当前 Demo 使用演示数据展示统计效果：`单个 = 20`、`叠放 = 20`，总样本数为 40。
   - 统计窗口打开后会自动调用 AI，对类别分布、潜在异常和改进建议进行分析。

7. **统计报告 PDF 导出**
   - 在结果统计窗口中新增 `结果导出` 按钮。
   - 支持将统计摘要、柱状图、扇形图、统计明细和 AI 分析结果导出为 PDF 文档。
   - PDF 采用白底报告样式，适合归档、展示或提交。

通过以上工作，本 Demo 不再只是一个简单的视觉流程运行示例，而是形成了一个“视觉流程运行 + AI 辅助开发 + 数据统计分析 + 报告导出”的完整闭环。

## 二、创新点与效率提升故事

### 2.1 创新点一：将 AI 助手嵌入机器视觉开发现场

传统机器视觉开发通常依赖工程师在多个工具之间来回切换：一边打开 VisionMaster 调试流程，一边查 SDK 手册，一边搜索错误信息，一边记录测试结果。本 Demo 将 AI 助手直接嵌入到 VisionMaster 控制界面旁边，让开发者在运行和调试过程中可以即时提问。

例如，当流程运行失败或接口调用报错时，工程师不需要离开当前程序，只需要把错误信息输入 AI 面板，AI 就可以结合上下文给出排查建议。

### 2.2 创新点二：基于 VM 手册的 RAG 检索增强

普通大模型可能并不了解 VisionMaster SDK 的具体接口细节，回答容易泛化。本 Demo 加入了本地 VM 手册 RAG 能力：

- 自动解析 VisionMaster CHM 帮助手册。
- 将手册内容切分成知识片段。
- 根据用户问题检索最相关的手册内容。
- 将检索结果注入到 AI 请求中。

这样 AI 的回答不再完全依赖模型记忆，而是可以基于本地官方资料回答问题，提高准确性和可信度。

### 2.3 创新点三：从运行结果到智能分析报告的一键闭环

Demo 不仅能运行视觉流程，还能在连续运行结束后展示结果统计，并自动调用 AI 生成分析结论。最终，用户可以将统计图表和 AI 分析导出成 PDF 报告。

这相当于把原本需要人工整理的“测试结果记录、图表截图、问题分析、报告输出”串成了一条自动化流程。

### 2.4 创新点四：面向工程开发的轻量化集成

本 Demo 基于 .NET Framework 4.8 WinForms 实现，不依赖复杂的 Web 服务部署。程序可以直接在 Windows 工控机或开发机上运行，符合机器视觉项目常见的交付环境。

PDF 导出没有引入额外第三方库，而是通过 GDI+ 绘制报告页面并生成标准 PDF 文件，降低部署复杂度。

### 2.5 效率提升小故事

在一次产线视觉检测项目中，团队需要调试一个“单个工件”和“叠放工件”的分类检测流程。以往的工作方式是：

1. 工程师先在 VisionMaster 中运行流程。
2. 运行结束后人工记录每一类的数量。
3. 手动截图柱状图或整理 Excel 表格。
4. 遇到 SDK 报错时再去翻 CHM 手册或搜索资料。
5. 最后根据经验写一段测试分析报告。

整个过程看似简单，但每次测试都要重复整理数据，效率并不高。尤其当现场节奏紧张时，工程师很容易漏记数据，或者在多个窗口之间切换时打断思路。

使用本 Demo 后，流程发生了变化：

- 工程师加载 `.sol` 方案后，直接在 Demo 中选择流程并连续运行。
- 运行结束后点击 `结果统计`，系统立即展示类别数量、占比、柱状图和扇形图。
- AI 自动分析当前分布，指出两类样本数量均衡、当前演示数据未显示明显偏置，同时建议后续采集更多边界样本验证模型稳定性。
- 工程师点击 `结果导出`，几秒钟内就得到一份 PDF 报告。
- 如果中途遇到 VisionMaster 接口或参数问题，直接在右下角 AI 助手中提问，AI 会结合 VM 手册给出说明。

原本需要十几分钟整理的测试记录，现在可以在一两分钟内完成。更重要的是，AI 助手减少了查资料和反复试错的时间，让工程师能把精力更多放在方案优化本身。

## 三、使用技术、实现方式、框架接口与运行环境

### 3.1 开发语言与应用框架

本 Demo 使用以下技术栈：

| 类别 | 技术 |
|---|---|
| 开发语言 | C# |
| 桌面框架 | Windows Forms |
| 运行框架 | .NET Framework 4.8 |
| 视觉平台 | HikRobot VisionMaster EDU 4.3.0 SDK |
| AI 接口 | DeepSeek / OpenAI-compatible Chat Completions API |
| 网络通信 | `HttpClient` |
| JSON 处理 | `JavaScriptSerializer` |
| 图形绘制 | GDI+ / `System.Drawing` |
| 配置管理 | `App.config` / `ConfigurationManager` |
| PDF 导出 | 自定义 PDF 写入逻辑 + JPEG 页面嵌入 |

### 3.2 VisionMaster 集成实现

Demo 通过 VisionMaster SDK 实现方案加载、流程控制和事件监听，关键能力包括：

- 加载 `.sol` 方案。
- 获取流程列表。
- 执行单次运行。
- 启动和停止连续运行。
- 接收流程开始、停止、工作状态等事件。
- 使用 `VmRenderControl` 显示图像渲染结果。
- 使用 `VmGlobalToolControl`、`VmMainViewConfigControl` 等控件支持参数配置。

主窗体 `MainForm` 负责整体界面调度，包括：

- 方案区域：选择、加载、保存方案。
- 流程区域：选择流程、运行一次、连续运行、结果统计。
- 显示区域：渲染视图和配置视图切换。
- AI 区域：嵌入 `ChatPanel` 聊天控件。

### 3.3 AI 聊天功能实现

AI 聊天功能由 `ChatPanel`、`DeepSeekClient`、`ChatMessage` 和 `AppSettings` 共同完成。

实现流程如下：

1. 用户在聊天面板输入问题。
2. `ChatPanel` 将用户消息加入上下文。
3. 系统调用 `VmManualRagService`，检索 VM 手册片段。
4. 将检索到的上下文和用户问题组合成增强后的请求。
5. `DeepSeekClient` 调用 `/v1/chat/completions` 接口。
6. 使用 Server-Sent Events 流式接收模型回复。
7. 回复内容实时显示在聊天窗口中。

AI 参数可在设置窗口中配置，包括：

- API Key
- Base URL
- 模型名称
- Temperature
- Max Tokens
- Timeout
- System Prompt

默认 Base URL 为：

```text
https://api.deepseek.com
```

常用模型包括：

```text
deepseek-chat
deepseek-reasoner
deepseek-coder
```

### 3.4 VM 手册 RAG 实现

RAG 模块由 `VmManualRagService` 实现，主要步骤如下：

1. 定位本地 `HikRobotVMHelp.chm` 手册文件。
2. 将 CHM 文件复制到本地缓存目录。
3. 使用 Windows `hh.exe` 反编译 CHM 内容。
4. 遍历提取出的 HTML 文件。
5. 清洗文本并切分为知识片段。
6. 建立本地索引文件 `index.json`。
7. 用户提问时，根据关键词和文本相关度检索 TopK 片段。
8. 将片段注入到最后一条用户消息中。

缓存目录默认位于：

```text
%LOCALAPPDATA%\ChatDemoCs\VmManualRag
```

该设计避免了每次启动都重新解析手册，提高了运行效率。

### 3.5 结果统计与 AI 分析实现

结果统计由 `ResultStatistics` 和 `ResultStatisticsForm` 实现。

`ResultStatistics` 负责：

- 保存标签计数。
- 记录总样本数。
- 记录流程名称。
- 提供线程安全的统计读写。
- 提供演示统计数据。

`ResultStatisticsForm` 负责：

- 展示统计摘要。
- 绘制柱状图。
- 绘制扇形图。
- 展示明细表。
- 自动调用 AI 分析。
- 支持复制、清空、关闭和导出。

AI 分析请求中会将标签、数量和占比组织成文本，要求 AI 从以下角度回答：

1. 分布特征。
2. 潜在异常。
3. 工程改进建议。

### 3.6 PDF 导出实现

PDF 导出功能在统计窗口中完成，导出内容包括：

- 报告标题。
- 流程名称。
- 总样本数。
- 类别数。
- 导出时间。
- 白底柱状图。
- 白底扇形图。
- 统计明细表。
- AI 分析文本。

实现方式为：

1. 使用 GDI+ 将报告页面绘制到位图中。
2. 将页面位图编码为 JPEG。
3. 按 PDF 1.4 格式写入对象、页面、图像流、交叉引用表和 trailer。
4. 生成可直接打开的 `.pdf` 文件。

这种方式避免了引入 PDF 第三方依赖，更适合 Demo 的轻量化交付。

### 3.7 运行环境

推荐运行环境如下：

| 项目 | 要求 |
|---|---|
| 操作系统 | Windows 10 / Windows 11 |
| .NET 运行时 | .NET Framework 4.8 |
| 开发工具 | Visual Studio 2022 或更高版本 |
| 编译工具 | MSBuild |
| VisionMaster | VisionMaster_EDU4.3.0 SDK |

Release 输出路径：

```text
D:\ApplicationDemo\ChatDemoCs\bin\Release\ChatDemoCs.exe
```

## 四、使用教程与常见错误处理

### 4.1 编译与启动

#### 方式一：使用 Visual Studio

1. 打开解决方案：

```text
D:\ApplicationDemo\ApplicationDemo.sln
```

2. 选择 `ChatDemoCs` 项目。
3. 右键设置为启动项目。
4. 点击 `生成解决方案`。
5. 按 `F5` 运行。

#### 方式二：使用 MSBuild

在 PowerShell 中执行：

```powershell
& 'C:\Program Files\Microsoft Visual Studio\18\Community\MSBuild\Current\Bin\MSBuild.exe' `
    'D:\ApplicationDemo\ChatDemoCs\ChatDemoCs.csproj' `
    /p:Configuration=Release `
    /p:Platform=AnyCPU
```

编译成功后运行：

```text
D:\ApplicationDemo\ChatDemoCs\bin\Release\ChatDemoCs.exe
```

### 4.2 VisionMaster 基本使用步骤

1. 启动程序。
2. 在 `Solution` 区域点击 `Select`。
3. 选择 VisionMaster `.sol` 方案文件。
4. 点击 `Load` 加载方案。
5. 在流程下拉框中选择需要运行的流程。
6. 点击 `Run Once` 执行一次。
7. 点击 `Run Continuous` 开始连续运行。
8. 再次点击停止连续运行。
9. 点击 `Render` 查看图像显示结果。
10. 点击 `Config` 进入参数配置界面。
11. 点击 `Save` 保存方案。

### 4.3 AI 助手使用步骤

1. 在右下角聊天面板点击 `Settings`。
2. 填写 API Key。
3. 确认 Base URL，例如：

```text
https://api.deepseek.com
```

4. 选择模型，例如：

```text
deepseek-chat
```

5. 根据需要调整 Temperature、Max Tokens、Timeout。
6. 点击 `Save` 保存配置。
7. 在输入框中输入问题。
8. 点击 `Send` 或按 `Ctrl + Enter` 发送。
9. 如需停止当前回复，点击 `Stop`。
10. 如需导出聊天记录，点击 `Export`。

### 4.4 结果统计与 PDF 导出步骤

1. 加载方案并选择流程。
2. 点击 `Run Continuous` 开始连续运行。
3. 停止连续运行。
4. 点击 `结果统计`。
5. 在弹窗中查看：
   - 总样本数。
   - 类别数。
   - 柱状图。
   - 扇形图。
   - 统计明细。
   - AI 分析。
6. 点击 `结果导出`。
7. 选择保存位置。
8. 导出 `.pdf` 报告。

### 4.5 常见错误与处理方式

| 问题 | 可能原因 | 处理方式 |
|---|---|---|
| 无法加载 `.sol` 文件 | 路径不存在、文件损坏、VisionMaster SDK 未安装 | 确认 `.sol` 路径有效，并检查 VisionMaster 是否正常安装 |
| 找不到 VM SDK 程序集 | SDK 安装路径与项目引用路径不一致 | 检查 `C:\Program Files\VisionMaster_EDU4.3.0` 是否存在，必要时修改 `.csproj` 引用 |
| 点击运行无反应 | 未加载方案或未选择流程 | 先点击 `Load`，再选择流程 |
| 连续运行后按钮状态异常 | 流程状态事件未正确返回或流程异常停止 | 查看右侧日志，确认 VisionMaster 流程是否正常结束 |
| AI 提示 API Key 为空 | 未配置密钥 | 打开 `Settings` 填写 API Key |
| AI 返回 HTTP 401 | API Key 错误或无权限 | 检查密钥是否正确、是否过期 |
| AI 返回 HTTP 404 | Base URL 或接口路径错误 | DeepSeek 默认 Base URL 使用 `https://api.deepseek.com` |
| AI 请求超时 | 网络不稳定、代理异常、模型响应慢 | 增大 Timeout，检查网络或代理设置 |
| RAG 提示找不到 CHM 文件 | VM 手册文件不存在或路径错误 | 确认 `Manuals\HikRobotVMHelp.chm` 存在，或在配置中指定正确路径 |
| RAG 加载失败 | CHM 反编译失败、缓存目录不可写 | 检查 `%LOCALAPPDATA%\ChatDemoCs\VmManualRag` 权限，可删除缓存后重试 |
| PDF 导出失败 | 目标文件被占用或目录不可写 | 关闭已打开的 PDF，换一个可写目录重新导出 |
| 中文显示异常 | 系统缺少中文字体或字体渲染异常 | 确认系统安装微软雅黑等中文字体 |
| 配置无法保存 | 程序运行目录无写权限 | 将程序复制到用户可写目录运行，或以适当权限启动 |

### 4.6 使用注意事项

1. 不要将真实 API Key 写入公开仓库或分发给无关人员。
2. VisionMaster 方案运行依赖本机 SDK、授权和运行环境。
3. AI 功能需要网络访问 DeepSeek 或兼容 API 服务。
4. RAG 手册问答依赖本地 CHM 手册文件。
5. PDF 导出时，如果目标 PDF 正在被浏览器或阅读器打开，可能会因为文件占用导致保存失败。
6. 建议在演示前提前配置好 API Key，并测试一次 AI 回复和 PDF 导出功能。

## 五、总结

本 Demo 将传统机器视觉流程控制与 AI 能力结合起来，完成了从方案加载、流程运行、参数调试、智能问答、结果统计、AI 分析到 PDF 报告导出的完整链路。

它的价值主要体现在三个方面：

1. **提升开发效率**：减少查手册、查接口、整理结果和写报告的重复劳动。
2. **降低使用门槛**：通过 AI 助手帮助开发者理解 VisionMaster 接口和错误信息。
3. **增强交付能力**：通过统计图表和 PDF 报告，让测试结果更直观、更规范。

因此，该作品不仅是一个功能演示程序，也可以看作是机器视觉开发工作流智能化的一次探索。
